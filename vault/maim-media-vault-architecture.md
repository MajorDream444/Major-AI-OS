# MAIM Media Vault Architecture

Status: Active architecture
Date: 2026-06-26

## Core Doctrine

MAIM is not only creating content.

MAIM is creating owned intellectual property.

Every live show, lesson, story, prompt, clip, transcript, article, workbook, deck, challenge, and merchandise concept becomes part of the MAIM IP library.

The Vault is where MAIM protects, organizes, repurposes, and monetizes that IP.

## Vault Purpose

The MAIM Media Vault exists to turn weekly teaching into permanent assets.

Each episode should become a reusable IP package that can power:

- YouTube
- Substack
- Podcast
- Instagram
- LinkedIn
- X / Threads
- Community prompts
- Course lessons
- Workbooks
- Prompt packs
- Membership content
- Merch concepts
- Future app content
- Future licensing products

## Source-of-Truth Rule

Google Drive stores heavy media files and visual exports.

GitHub stores:

- prompts
- docs
- scripts
- episode manifests
- workflow definitions
- reusable templates
- source-of-truth summaries

Airtable tracks production status.

Notion stores working summaries and planning pages.

## Episode Manifest Standard

Each episode should eventually have a GitHub manifest:

```txt
vault/episodes/episode-001-ai-mindset/manifest.md
```

A manifest should track:

- episode title
- teaching spine
- Drive folder
- YouTube URL
- Substack URL
- NotebookLM video
- transcript
- social clips
- captions
- challenge
- merch concepts
- analytics
- lessons learned
