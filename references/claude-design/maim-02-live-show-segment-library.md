## Core Reframe

MAIM 02 is evolving from a standard webinar into a recurring edutainment show.

The show should still teach clearly.

It should still convert.

But it should not feel like a dry webinar that only leads to a pitch.

It should feel like:

```txt
A weekly live learning show
A confidence room
A future translation room
A community classroom
An edutainment engine
```

Working show identity:

```txt
Major AI Mindset Live
```

Working episode format:

```txt
AI Mindset: See It. Believe It. Build It.
```

## Operating Philosophy

Each week should improve the format.

By week 15 or 16, the show should feel evolved, not identical to week 1.

The letters change, but the experience should also mature through better segments, stronger pacing, reusable show pieces, recurring characters, better visuals, NotebookLM-style videos, audience moments, community spotlights, and eventually Remotion-powered opening and closing sequences.

## Weekly Frequency

Target:

```txt
Weekly
```

Minimum operating standard:

```txt
3 out of 4 weeks per month
```

If a live week is skipped, use a replay, vault drop, or pre-recorded segment to keep the rhythm alive.

## 60-Minute Live Show Format

The weekly show should be planned for 60 minutes.

Recommended format:

```txt
0–5 min      Opening + grounding
5–10 min     Major's opening frame / founder note
10–18 min    Lesson Block 1
18–22 min    NotebookLM-style AI News / Roundtable Segment
22–32 min    Lesson Block 2
32–37 min    Live demo / interaction / chat exercise
37–47 min    Lesson Block 3
47–52 min    Builder story / visual explainer / recap segment
52–57 min    Weekly challenge + CTA
57–60 min    Closing reflection + next episode teaser
```

The goal is to change the pattern every 10 to 15 minutes so people stay engaged without feeling rushed.

## Reusable Segment Library

### 1. Opening Sequence

Purpose:

```txt
Set energy, signal quality, establish the world.
```

Future tool:

```txt
Remotion
```

Possible ingredients:

- Crown coin animation
- MAIM sonic tag
- Tagline
- Quick episode title
- Warm founder entrance

### 2. Major's Opening Frame

Purpose:

```txt
Ground the room and give the episode a human reason to exist.
```

Tone:

```txt
Calm, direct, warm, personal, not hype.
```

### 3. ABC Lesson Block

Purpose:

```txt
Teach one MAIM concept in plain language.
```

Each block includes:

```txt
One concept
One example
One five-minute action
One memory hook
```

### 4. NotebookLM-Style AI News Segment

Purpose:

```txt
Make the future feel understandable and current.
```

Length:

```txt
2–3 minutes initially
4–5 minutes when the format matures
```

Use cases:

- AI news explained simply
- What changed this week
- What builders should pay attention to
- What not to panic about
- One practical implication for everyday people

Style:

```txt
Two-to-five-host conversation
Calm, conversational, entertaining, beginner-safe
```

Storage:

```txt
Upload final videos to YouTube or Drive.
Reference scripts and show notes in GitHub.
```

### 5. Live Demo Segment

Purpose:

```txt
Turn belief into proof.
```

Examples:

- Ask AI a vague question, then ask with context
- Build a content idea live
- Turn a repeated task into an automation map
- Draft a simple offer
- Create a weekly action plan

### 6. Community Interaction Segment

Purpose:

```txt
Make the room feel alive.
```

Formats:

- Chat question
- Poll
- Type one task you repeat every week
- Share one thing you want AI to help with
- Reply with your business / project lane

### 7. Builder Spotlight

Purpose:

```txt
Show real people using AI in real life.
```

Future examples:

- Barber using AI for follow-up
- Pastor turning sermons into content
- Coach building a 7-day product
- Parent helping a child learn
- Athlete building NIL content

### 8. Visual Explainer

Purpose:

```txt
Break up slides with a short story, animation, or metaphor.
```

Possible tools:

- Google Omni / Flow
- Remotion
- NotebookLM / Open Notebook
- Voicebox
- AI Voice Generator

### 9. Weekly Challenge

Purpose:

```txt
Move people from inspiration into action.
```

Example:

```txt
Before next week, write down three repeated tasks and ask AI how to simplify one of them.
```

### 10. CTA / Invitation Segment

Purpose:

```txt
Invite without making the whole show feel like a sales trap.
```

Early phase CTA:

```txt
Join the list / community.
Show up next week.
Complete the weekly challenge.
```

Later monetization CTA:

```txt
Join the vault.
Join the 7-Day Digital Asset Lab.
Join the 4-Week Leverage Cohort.
Buy membership pins or starter kits.
```

### 11. Replay / Vault Segment

Purpose:

```txt
Turn live sessions into recurring assets.
```

Rules:

- Live episode is free during the early build phase.
- Replay may run once per week.
- Archived episodes and resources eventually move into the MAIM Vault.
- Vault access can become a paid membership tier.

### 12. Closing Sequence

Purpose:

```txt
Create emotional completion and a reason to return.
```

Closing line:

```txt
Likkle by likkle, we build the future.
```

Future tool:

```txt
Remotion-powered outro
```

## Monetization Doctrine

MAIM Live starts free to build trust and audience rhythm.

But MAIM is not a charity-only content stream.

The system should be designed to monetize ethically through:

```txt
Paid vault access
Community membership tiers
7-Day Digital Asset Lab
4-Week Leverage Cohort
Membership pins / physical products
Shopify store
Digital templates and prompt packs
Replay access
Sponsorships later
```

The first phase should build trust before pushing hard monetization.

But every show should still have a next step, a sign-up path, and a way to identify serious builders.

## Audience Growth Target Logic

Initial benchmark:

```txt
5,000 followers → target at least $5,000/month in consistent revenue
```

This does not happen automatically.

It requires:

- consistent show rhythm
- lead capture
- segmented follow-up
- community offers
- replay/vault system
- clear product ladder
- trust before pressure
