## Canonical Beginner Spine

The first five letters are locked as:

```txt
A = Awareness
B = Belief
C = Context
D = Direction
E = Experiment
```

## Why E = Experiment

Experiment is the right fifth letter because MAIM teaches people to test before they overbuild.

The first movement is:

```txt
Awareness → Belief → Context → Direction → Experiment
```

That means:

1. Notice the opportunity.
2. Believe you can begin.
3. Give the tool better context.
4. Aim the work at a clear direction.
5. Run a small experiment.

Experiment keeps beginners from getting trapped in theory, perfection, or tool-chasing.

It teaches them to try one small thing, observe what happens, and improve the next attempt.

## Execution Still Exists

Execution is still part of MAIM.

It is currently assigned to:

```txt
X = eXecution
```

Reason:

Execution is the shipping layer.

Experiment is the testing layer.

This separation matters.

Experiment asks:

```txt
What can I test?
```

Execution asks:

```txt
What can I ship?
```

## First Five Letter Teaching Notes

### A = Awareness

Notice before you automate.

Look for time leaks, repeated tasks, attention drains, friction, and places where help is needed.

### B = Belief

If you can text, you can use AI.

Belief lowers the barrier so people stop waiting for technical permission.

### C = Context

Clear input creates clearer output.

AI does not read minds. Context turns vague requests into usable results.

### D = Direction

Aim AI at a goal.

A tool without direction becomes noise.

### E = Experiment

Run one small test.

AI confidence grows when people test, observe, adjust, and try again.

## Final Rule

The ABCs should always move people from idea to action without overwhelming them.

One letter.

One lesson.

One action.
