# MAIM Multimodal Memory Layer

Status: Active doctrine
Related project: `MAIM 03 ABC Roadmap / Learning Engine`
Related workflow: `workflows/weekly-webinar-build-loop.md`

## Purpose

The MAIM Learning Engine should not only teach through text and slides.

It should help people remember through multiple learning channels:

- Visual memory
- Audio memory
- Spoken memory
- Written memory
- Action memory
- Community memory

The ABCs become the curriculum.
The webinars become the classroom.
The community becomes the practice field.
The songs become the memory layer.
Obsidian becomes the knowledge graph.
GitHub remains the source of truth.
Claude Design remains the visual factory.

## Core Doctrine

Every important lesson should be able to become:

```txt
See it
Hear it
Say it
Write it
Practice it
Share it
Remember it
```

People learn differently.

Some people need a visual metaphor.
Some need a story.
Some need a song.
Some need a worksheet.
Some need a repeated mantra.
Some need a community conversation.

MAIM should respect all of those paths.

## Memory Hook Standard

Every ABC lesson should include a memory hook.

A memory hook is a short phrase that becomes easy to repeat.

Examples:

```txt
A — Awareness: You cannot automate what you have not noticed.
B — Belief: Belief comes before skill.
C — Context: Context creates clarity.
D — Direction: Direction beats speed.
E — Experiment: Progress belongs to the people who experiment.
```

These hooks should appear in:

- Instagram captions
- Webinar slides
- Reel scripts
- Podcast intros
- Songs
- Workbook pages
- Community prompts
- Course summaries

## Signature Metaphor Standard

Every ABC lesson should have one strong metaphor.

The metaphor should be simple enough for a beginner to understand in three seconds.

Examples:

```txt
A — Awareness: leaking bucket / glowing repeat pattern
B — Belief: first domino / unlocked doorway
C — Context: camera focus / blurry-to-clear lens
D — Direction: compass / single road
E — Experiment: small lab card / try-learn-adjust loop
```

Metaphors should start as striped placeholders or simple graphic marks in Claude Design.

Later they can become:

- Cinematic images
- Storybook illustrations
- Motion graphics
- Reel animations
- Workbook icons
- Song visuals

## Sonic Identity

MAIM should develop a sonic identity the same way it develops a visual identity.

The sound should be:

- Warm
- Familiar
- Uplifting
- Caribbean-rooted
- Beginner-safe
- Easy to remember
- Not childish unless intentionally made for kids

Possible styles:

- Reggae learning chant
- Afrobeats educational hook
- Lo-fi study groove
- Gospel call-and-response
- Acoustic campfire version
- Hip-hop memory verse
- Kids storybook version
- Webinar transition sting
- Meditation / sanctuary version

## Song System

Each ABC concept can eventually have:

```txt
1. Adult hook
2. Kids version
3. Instrumental loop
4. 15-second reel sound
5. Webinar transition sting
6. Community chant / call-and-response
7. Audio mnemonic
```

The goal is not to copy a specific artist.

The goal is to borrow the principle of familiarity: simple melodies, repeatable lines, and emotional resonance.

## Obsidian Role

Obsidian is the private context and knowledge graph layer.

Use Obsidian to store:

- Lesson notes
- Founder reflections
- Raw ideas
- Song fragments
- Webinar insights
- Community questions
- Voice memo transcripts
- Metaphor experiments
- Personal stories
- Version history of concepts

GitHub stores the public operating system.

Obsidian stores the living thought process.

## Repo Role

When an idea becomes reusable, stable, or operational, move it from Obsidian into GitHub.

Examples:

- A song fragment stays in Obsidian.
- A finished sonic identity brief goes into GitHub.
- A private reflection stays in Obsidian.
- A teaching module goes into GitHub.
- A messy metaphor list stays in Obsidian.
- A final metaphor map goes into GitHub.


This turns MAIM from a content system into a memory system.

## Example: A is for Awareness

Visual memory:

```txt
A leaking bucket or glowing repeat pattern showing where time leaks.
```

Memory hook:

```txt
You cannot automate what you have not noticed.
```

Song hook:

```txt
A is for Awareness, notice what repeats.
Every little leak becomes a clue we need.
```

Action memory:

```txt
Write down three tasks that drain your time each week.
```

Community memory:

```txt
What did you notice today that keeps repeating?
```
