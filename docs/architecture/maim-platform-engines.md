# MAIM Platform Engines

Status: foundation architecture
Owner: Major AI Mindset

The internal platform model is engine-based.

Each engine should have clear inputs, outputs, release gates, and agent ownership.

## Engine Map

```txt
MAIM Platform
├── Command Room
├── Knowledge Engine
├── Registration Engine
├── Content Engine
├── Agent Engine
├── Community Engine
├── Analytics Engine
└── Legacy Engine
```

## Command Room

Purpose:

The front door into MAIM.

Owns:

- landing page
- next session
- public trust
- registration entry
- Google Form fallback

## Knowledge Engine

Purpose:

What MAIM teaches.

Owns:

- ABC lessons
- knowledge sessions
- prompt packs
- stories
- case studies
- playbooks

Primary folder:

```txt
knowledge/
```

## Registration Engine

Purpose:

Move people safely from interest into the operating system.

Owns:

- Apps Script intake
- Google Sheet source log
- Airtable sync gate
- dry-run previews
- pipeline events
- duplicate protection

## Content Engine

Purpose:

Turn MAIM knowledge into media and reusable assets.

Owns:

- YouTube
- thumbnails
- lead magnets
- ebooks
- Remotion templates
- newsletter / Substack paths

## Agent Engine

Purpose:

Route work and follow-up through specialized agents.

Owns:

- agent playbooks
- agent routing
- lane-aware follow-up
- future agent marketplace

## Community Engine

Purpose:

Turn sessions into relationships, cohorts, and ongoing participation.

Owns:

- cohorts
- discussions
- member paths
- facilitator workflows
- future global chapters

## Analytics Engine

Purpose:

Show what is working.

Owns:

- registration metrics
- attendance
- completion
- content performance
- funnel movement
- agent effectiveness

## Legacy Engine

Purpose:

Preserve, license, and expand the MAIM knowledge system.

Owns:

- knowledge vault
- licensing
- certification
- artifacts
- future tokenization
- long-term IP structure
