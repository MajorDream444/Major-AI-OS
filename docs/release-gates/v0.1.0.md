# MAIM Release Gate - v0.1.0

Status: shipped and tagged
Release date: 2026-07-06
Primary system: MAIM Command Room landing page

## What Changed

- Shipped the MAIM Command Room landing page refinement as the stable visual baseline.
- Kept the app inside `experiments/landing-pages/google-ai-studio/maim-mission-control-vite`.
- Added root `.gitignore` housekeeping so `node_modules/`, `dist/`, media exports, `.DS_Store`, and env files stay out of git.
- Preserved the registration form wiring to the Apps Script webhook and the Google Form backup path.

## What Was Tested

Level 1 - Code Quality:

- `npm install`
- `npm run lint`
- `npm run build`

Level 2 - Logic:

- Registration form compiled.
- Hero rendered without missing image assets.
- Google Form fallback remained visible.
- Mobile layout was checked for obvious horizontal overflow.

Level 3 - Platform:

- Not part of this release gate. Apps Script intake was still a follow-on system.

Level 4 - Production:

- Main branch verified after merge.
- Release tag created.

## What Is Still Disabled

- Airtable sync.
- Live welcome email.
- Calendar invite.
- ManyChat.
- CRM write.

## What Is Safe

- Landing page visual experience.
- Local build process.
- Static content fallback.
- Registration form UI and backup path.

## What Is Experimental

- The Apps Script webhook was connected from the landing page, but the registration onboarding engine had not yet passed the full live release gate.
- Runtime CMS endpoint remained configurable and fallback-driven.

## Rollback Procedure

1. Revert the relevant landing-page merge commit if a code issue appears.
2. Redeploy the previous Vercel production deployment.
3. Keep the Google Form fallback available while the landing page is corrected.
