## What Is Still Disabled

- Airtable sync.
- Live welcome email.
- Calendar invite.
- ManyChat.
- CRM write.

These remain intentionally disabled for Sprint 001.

## What Is Safe

- A registration can enter the system in dry-run mode.
- The source Google Sheet can receive the row.
- The welcome email can be previewed without sending.
- Pipeline events can prove what happened.
- Duplicate submissions are blocked for the same email and session.

## What Is Experimental

- Airtable sync is the next subsystem and must remain dry-run-first in Sprint 002.
- Welcome email live sending is reserved for Sprint 003.
- Calendar invite, ManyChat, and CRM writes remain future gated subsystems.

## Rollback Procedure

1. Set Apps Script `PIPELINE_MODE=registration_sprint`.
2. Keep `DRY_RUN_MODE=true`.
3. Disable or remove the landing-page webhook environment variable if intake must pause.
4. Keep the Google Form fallback visible.
5. Revert PR #18 if the script itself needs to be removed from `main`.
