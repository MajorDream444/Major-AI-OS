# MAIM Design Philosophy

Status: canonical
Owner: Major AI Mindset

This document defines the design philosophy behind the MAIM Command Room and future MAIM interfaces.

## Principle 1: Invitation, Not Advertisement

The landing page is not an advertisement.

It is an invitation.

The visitor should feel they are being welcomed into a serious knowledge room, not pressured by a campaign poster.

## Principle 2: Calm Is a Feature

The visitor should feel calmer after arriving than before.

MAIM should reduce confusion, not add noise. Every page should lower the emotional cost of taking the next step.

## Principle 3: Reveal, Do Not Overwhelm

Never overwhelm.

Reveal.

The page should lead people from curiosity into clarity through rhythm, hierarchy, and restraint.

## Principle 4: Teach Before Selling

Every major surface should teach one useful thing before it asks for commitment.

Trust is built through usefulness.

## Principle 5: People Enter a Universe

People do not buy software first.

They enter a universe.

The interface should make MAIM feel like a world of lessons, sessions, prompts, agents, systems, and human transformation.

## Principle 6: The Human Is the Hero

The human is always the hero.

AI is the amplifier.

The product should make people feel more capable, not replaced.

## Principle 7: Five-Second Clarity

Every page should answer within five seconds:

```txt
What is this?
Who is it for?
What should I do next?
```

If the page cannot answer those questions quickly, it is not finished.

## Principle 8: Mystery to Clarity

The visual language moves from mystery to clarity:

```txt
Portal
-> Understanding
-> Action
-> Systems
-> Legacy
```

That progression mirrors the MAIM journey itself.

## Command Room Model

Stop thinking of the first public experience as only a landing page.

Call it the MAIM Command Room.

```txt
Command Room
-> Knowledge Sessions
-> ABC Library
-> Discovery Calls
-> Prompt Library
-> Labs
-> Agent Marketplace
-> Legacy OS
```

The landing page is the front door. The Command Room is the operating model.

## Design Test

Before approving a MAIM page, ask:

- Does this feel like an invitation instead of an ad?
- Does it make the visitor calmer?
- Does it reveal instead of overwhelm?
- Does it teach before selling?
- Does it make the human the hero?
- Can someone understand what to do next within five seconds?
