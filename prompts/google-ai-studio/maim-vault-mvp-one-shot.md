# Google AI Studio Prompt — MAIM Vault MVP One-Shot Prototype

Use this prompt in Google AI Studio / Gemini to create a quick working visual prototype.

```txt
Build a one-shot prototype for a web app called MAIM Vault.

MAIM Vault is the internal knowledge and asset operating system for Major AI Mindset.

The Vault stores, organizes, and tracks all intellectual property created from webinars, Knowledge Vault Sessions, ABC lessons, prompts, videos, images, courses, agents, and documents.

Core formula:
Idea → Asset → IP → Vault → Reuse → Revenue

Core purpose:
Help the founder and team see what assets exist, where they live, what status they are in, and what should happen next.

Design style:
Premium dark command center.
Black foundation.
Gold authority.
Jamaican green legacy.
Champion blue action energy.
Orange CTA accents.
White clarity.
Editorial dashboard style.
Calm operator energy.
No generic SaaS look.
No cartoon robots.
No clutter.

Use the tagline:
Likkle by likkle, we build the future.

Build these screens:

1. Dashboard
Show cards for:
- Total Assets
- Knowledge Sessions
- Ready to Publish
- Needs Review
- Published
- Archived

Also show recent activity and next actions.

2. Asset Library
Create a searchable table or card grid with sample assets.
Fields:
- Asset ID
- Title
- Category
- Format
- Status
- Related Session
- Last Updated

3. Asset Detail Page
Show a detailed record for one asset.
Fields:
- Asset ID
- Title
- Description
- Category
- Format
- Status
- Version
- Owner
- Source Link
- Drive Link
- GitHub Path
- Notion Link
- Airtable Record
- Related Session
- Related ABC Letter
- Tags
- Revenue Potential
- Next Action

4. Knowledge Sessions
Show a list of Knowledge Vault Sessions.
Include this sample:
KVS-001 — The ABCs Are Life Principles First
Theme: Mindset before AI tools
Status: Indexed

5. Add New Asset
Create a simple form with:
- Title
- Category
- Format
- Status
- Source Link
- Tags
- Next Action

6. Status Board
Create a kanban board with columns:
Captured
Extracted
Indexed
Designed
Ready to Publish
Published
Archived

Sample assets:

Asset 1:
ID: MAIM-KVS-001-INDEX
Title: Knowledge Index — Session 001
Category: Knowledge Vault Session
Format: Markdown Document
Status: Indexed
Related Session: KVS-001
Tags: Awareness, Belief, Context, Mindset Operating System
Next Action: Extract short clips and carousel ideas

Asset 2:
ID: MAIM-ABC-A-001
Title: A — Awareness: Notice the Opportunity
Category: ABC Curriculum
Format: Image / Reel Asset
Status: Ready to Publish
Related Session: KVS-001
Tags: Awareness, ABC, Visual, Instagram
Next Action: Create caption and schedule post

Asset 3:
ID: MAIM-09-BOARD-001
Title: MAIM 09 — Webinar Knowledge Vault Board
Category: Design Board
Format: HTML
Status: Designed
Related Session: KVS-001
Tags: Vault, Webinar, IP, Knowledge OS
Next Action: Archive final HTML and extract diagrams

Asset 4:
ID: MAIM-PROMPT-001
Title: ABC Reframe Prompt
Category: Prompt
Format: Text
Status: Captured
Related Session: KVS-001
Tags: Prompt, ABC, Problem Solving
Next Action: Add to prompt library

Navigation:
Dashboard
Assets
Sessions
Status Board
Add Asset
Settings

The prototype can use local sample data. It does not need real authentication, database, or integrations yet.

Focus on clarity and speed.

The MVP should answer:
What do we have?
Where does it live?
What is its status?
What happens next?

Do not overbuild.
Make the Vault useful first.
```
