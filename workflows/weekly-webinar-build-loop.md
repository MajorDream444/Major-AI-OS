# Weekly Webinar Build Loop

Status: Active workflow
Related Claude Design project: `MAIM 03 ABC Roadmap`
Related funnel project: `MAIM 02 Webinar Funnel`

## Purpose

The weekly MAIM webinar is not only a presentation.

It is the community grounding room and curriculum feedback loop.

Each week should help beginners feel safer with AI, give them one clear concept, and move the curriculum forward.

## Core Rhythm

```txt
Build → Teach → Learn → Improve → Repeat
```

This replaces the old pressure pattern:

```txt
Build everything → Launch perfectly
```

MAIM grows by teaching in public, listening to the room, improving the lesson, and repurposing the best moments into assets.

## Weekly Operating Cadence

```txt
Monday      Build the lesson module
Tuesday     Design the visual assets
Wednesday   Write captions, scripts, and prompts
Thursday    Refine and approve
Friday      Test the webinar flow
Saturday    Prepare links, slides, and reminders
Sunday      Teach the live session
```

## Week 1 Webinar

Theme:

```txt
AI Mindset Grounding Session
```

Primary purpose:

```txt
Make AI feel safe, human, useful, and beginner-accessible.
```

Core letters:

```txt
A = Awareness
B = Belief
C = Context
```

Do not overload the room.

Week 1 should introduce:

- Why MAIM exists
- Who Major is
- Why AI is not just for tech people
- What the ABC roadmap is
- A, B, and C as the first doorway
- One five-minute action per letter
- How to join the community / next session

## Webinar Format

Target length: 60 minutes

Suggested structure:

```txt
0–5 min      Welcome + grounding
5–12 min     Founder story + why MAIM exists
12–20 min    What AI Mindset means
20–32 min    A is for Awareness
32–42 min    B is for Belief
42–52 min    C is for Context
52–58 min    Weekly action + community invitation
58–60 min    Closing: Likkle by likkle
```

## Teaching Rule

Every section should include:

```txt
One concept
One example
One small action
```

If a concept cannot be explained in plain language, it is not ready for the beginner webinar.

## Post-Webinar Capture

After each webinar, capture:

- Questions people asked
- Where people got confused
- Best quotes from Major
- Best audience reactions
- Best examples
- New objections
- Content clips to create
- Lesson updates needed

## Repurposing Outputs

Each webinar should produce:

- 3 short clips
- 1 carousel
- 1 caption
- 1 newsletter reflection
- 1 podcast topic
- 1 community prompt
- 1 lesson update

## Versioning Rule

Each ABC lesson should improve over time.

```txt
A v1 = first public teaching
A v1.1 = after first webinar questions
A v1.2 = after first content performance review
A v2 = when added to course or lead magnet
```

The goal is not perfection.

The goal is compounding clarity.

## Emotional Standard

The room should feel:

- Safe
- Grounded
- Included
- Curious
- Capable

The viewer should leave thinking:

```txt
I can start.
I am not too late.
I do not have to be technical.
I just need the next step.
```

## Closing Line

Use some version of:

```txt
Likkle by likkle, we build the future.
```
