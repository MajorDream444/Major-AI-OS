# MAIM Design Production Workflow

Brief → Design → Review → Export → Track → Publish → Archive → Improve

## Rule

Ready to Publish means the public asset is good enough to go live.

Complete means the full package is archived and tracked.

Do not let archive work block publishing.

## Brief

Every asset needs Asset ID, campaign, platform, format, headline, CTA, visual direction, export size, deadline, and status.

## Design

Design in Claude Design, Canva, Figma, Penpot, HTML templates, or another approved tool.

Claude Design projects should reference `MAIM 00 Brand System`.

## Review

Check five-second clarity, brand consistency, crown usage, beginner safety, CTA clarity, export readiness, and reuse potential.

## Export

Use canonical sizes from `data/design-tokens.json`.

## Track

Track in Airtable or fallback tracker with Asset ID, title, campaign, status, platform, format, file name, GitHub metadata path, Drive/local path, and notes.

## Publish

Publishing can happen through Meta Business Suite, Postiz, Instagram native, Skool, Substack, YouTube, or LinkedIn.

## Archive

Archive in Drive, GitHub metadata, local folder, and Obsidian when relationship context matters.

## Improve

Weekly, review what shipped, what got delayed, what templates were reused, and what should become a skill/plugin.
