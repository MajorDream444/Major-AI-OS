# Three-Tier ABC Production Workflow

## Purpose

This workflow operationalizes the Major AI Mindset Three-Tier ABC GTM strategy.

It is designed for Cowork, Codex, Claude Code, and human operators.

## Production Philosophy

Treat content as modular, upgradable software for the human mind.

Each letter should exist in three levels:

```txt
Beginner = safe first win
Medium   = repeatable workflow
Expert   = strategic system
```

## Production Lanes

### Lane 1 — Beginner Daily Content

Primary formats:

- 30–60 second reels
- Simple carousel
- Caption pack
- Five-minute action

Goal:

Mass reach, trust, and confidence.

### Lane 2 — Medium Weekly Depth

Primary formats:

- Carousel
- LinkedIn post
- Workflow worksheet
- Newsletter expansion
- Opal mini-app prompt

Goal:

Convert interested beginners into builders.

### Lane 3 — Expert Authority Layer

Primary formats:

- LinkedIn document
- Podcast episode
- Strategic memo
- Workshop outline
- Omni advanced video prompt

Goal:

Position Major AI Mindset as an AI strategy and operating system authority.

## Daily Operating Rhythm

### Morning — Operator Review

Cowork checks:

- Current letter status
- Airtable asset records
- Missing fields
- Approval blockers
- Publishing queue
- Comment/learning signals from previous posts

Output:

```txt
Daily MAIM Content Ops Brief
```

### Midday — Production / Build

Codex or human operator handles:

- Metadata updates
- GitHub files
- Export helpers
- App schema updates
- Prompt registry

### Evening — Community / Analytics

Cowork tracks:

- Comments
- Saves
- Shares
- DMs
- Learning signals
- Keyword comments
- CTA conversions

## Rolling Pipeline Model

Always keep three letters in motion:

```txt
Letter 1 = Approval / production
Letter 2 = Full package build
Letter 3 = Concept setup
```

Example:

```txt
A = Approval / publishing
B = Full package build
C = Concept setup
```

## Weekly Schedule Model

### Monday

Beginner reel / core letter drop.

### Tuesday

Carousel / caption repurpose.

### Wednesday

Medium workflow/deeper post.

### Thursday

Podcast / newsletter / community prompt.

### Friday

Expert LinkedIn document / strategic layer.

### Weekend

Review analytics and learning signals.

## Week 6 Consolidation Workflow

After Z, pause new letters for one week.

Create:

- A-Z recap reel
- A-Z carousel summary
- What we learned post
- Top 5 saved letters report
- Top 5 comment themes
- Medium tier teaser
- Expert waitlist CTA
- Community reflection prompt

## Repurposing Rules

### Instagram / TikTok

- Strong hook in first second
- Minimal text
- High contrast
- One clear action

### YouTube Shorts

- Searchable title
- Pinned prompt / action step
- Optional 60–90 second version

### LinkedIn

- Professional framing
- PDF document carousel when possible
- Leadership/workflow angle
- Less hype, more insight

### Newsletter

- Story
- Lesson
- Practical action
- Invitation

### Podcast

- Expand philosophy
- Use examples
- Connect tiers
- Tease next letter

## Cowork Responsibilities

Cowork should:

1. Run daily status checks.
2. Produce completion reports.
3. Flag missing assets.
4. Summarize learning signals.
5. Recommend next production action.
6. Never mark anything approved without Major.

## Codex Responsibilities

Codex should:

1. Maintain schemas.
2. Update GitHub files.
3. Build app utilities.
4. Add export helpers.
5. Keep package reports machine-readable.
6. Avoid changing GTM doctrine without instruction.

## Claude Code Responsibilities

Claude Code should:

1. Audit architecture.
2. Improve maintainability.
3. Review schema consistency.
4. Improve accessibility.
5. Review design-system alignment.
6. Suggest refactors without rewriting everything.

## Human Operator Responsibilities

Major approves:

- Concept decisions
- Visual direction
- Voice/tone
- Script approval
- Publishing approval
- Strategic pivots

## Definition of Done

A letter is complete only when:

- GitHub metadata is current
- Airtable records are current
- Notion review page is current
- Drive media masters are filed
- Captions/scripts/prompts are filed
- Status is updated
- Publishing/learning signals are reviewed
