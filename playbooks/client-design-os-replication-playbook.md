# Client Design OS Replication Playbook

This turns the MAIM setup into a repeatable client service.

Do not sell random graphics. Sell a visual operating system.

## Client Leaves With

- Source-of-truth repo or folder
- Brand system
- Design tokens
- Campaign templates
- Prompt library
- Production workflow
- Asset tracker
- Publishing-ready outputs
- Local archive

## Client Stack

GitHub = source of truth  
Claude Design = visual factory  
Claude Code / Codex = implementation  
Airtable / Baserow = tracker  
Drive / local storage = binary files  
Obsidian / Markdown = strategy memory  
n8n = automation  
Remotion = reusable video templates  
Vercel / self-host = deployment

## Intake

Ask for business name, offer, audience, visual references, logo, colors, voice samples, website, platforms, launch goal, CTA, deadline, budget, deliverables, and tools.

## Replication Steps

1. Create client folder or repo.
2. Create `README.md`, `CLAUDE.md`, `DESIGN.md`.
3. Create `data/design-tokens.json`.
4. Create Claude Design parent project.
5. Attach source files.
6. Upload visual references.
7. Generate brand system board.
8. Create campaign projects.
9. Reference parent project.
10. Generate templates.
11. Export assets.
12. Track in Airtable/Baserow.
13. Archive locally.
14. Deliver client package.

## Package Levels

Base: brand board, tokens, logo rules, 5 social templates, 3 campaign templates, prompt library, naming convention, export guide.

Advanced: landing page wireframe, lead magnet cover, thumbnail system, video prompts, Remotion direction, tracker, GitHub repo.

Premium: working site/app prototype, automated workflow, publishing schedule, asset dashboard, AI assistant prompts, self-host plan.

## Rule

Every client build should become easier than the last one.

Capture improvements. Turn them into templates. Turn templates into skills. Turn skills into plugins.

Likkle by likkle, we build the future.
