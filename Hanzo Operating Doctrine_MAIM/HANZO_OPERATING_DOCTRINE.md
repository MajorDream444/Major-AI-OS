## The Biggest Discovery: ABC Is a Lens, Not a Calendar

This happened during the first Knowledge Vault Session. Originally the
webinar was structured like:

```
Week 1 → A
Week 2 → B
Week 3 → C
```

**That was wrong.**

The breakthrough: every lesson teaches ABC. Every session teaches ABC.
Every conversation teaches ABC. Every problem teaches ABC.

**ABC is not chronological. ABC is a lens.**

This does not mean the weekly webinar abandons letter-based topic framing
— per MAIM 09 (Knowledge Vault), each week's live session is still tagged
to a single A–Z letter as its topic frame (e.g. `wk-031_b-belief`), and the
beginner A–E intro sequence on Instagram is intentionally sequential as a
front door. What changes is the *internal* application: Awareness, Belief,
and Context as concepts apply within whatever that week's content
naturally expresses — they are not three separate weeks of unrelated
material. Do not build content systems that treat A, B, and C as
sequential, mutually exclusive topics.

### ABC Doctrine

**Awareness** — Notice opportunity. See reality. Recognize patterns.
Recognize assumptions.

**Belief** — Believe change is possible. Believe you can build. Believe
you can learn. Believe your circumstances are not fixed.

**Context** — Context creates intelligence. AI needs context. Businesses
need context. Relationships need context. Life needs context. Everything
exists inside context.

**Never separate these.**

## Knowledge Vault Philosophy

Every conversation becomes IP. Every lesson becomes reusable. Every
recording becomes searchable. Every insight becomes connected.

```
Idea → Asset → IP → Vault → Reuse → Revenue
```

### The Vault Is Not Storage

This is critical. **Google Drive stores files. The Vault stores
knowledge.**

The Vault connects videos, transcripts, prompts, images, agents, courses,
design boards, articles, and revenue ideas — everything links together.
A folder of files is not a Vault. A Vault is a system where any piece of
content can be found, traced back to its source session, and reused
without starting from a blank page.

## Curriculum Source of Truth

The canonical, slot-by-slot content for every authored A–Z letter lives
in:

```
data/MAIM_LEARNING_ENGINE_CANONICAL.json
```

**This JSON file is the single source of truth for curriculum content.**
Reels, stories, carousels, workbook pages, course lessons, podcast
episodes, newsletters, and any future Remotion template must read module
content from this file — never hardcode a letter's question, mantra,
lesson, or action directly into a template or one-off content piece.

If a letter's canonical content needs to change, change it in the JSON
first. If a letter isn't yet in the JSON's `modules` object, its content
has not been authored — do not invent placeholder copy for it and ship it
as if it were canonical. Flag it as pending instead.
